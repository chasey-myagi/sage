# Agent Caster v0.0.1 — Scope

注册为 Rune Caster，跑通一条 Agent 任务。

## 目标

Caster 启动 → 注册 `agents.execute` rune → 接收 config + message → pi-mono Agent 执行 → 返回结果。

## 架构

```
Rune Runtime (gRPC :50070)
       │
       ▼
Agent Caster (本项目)
  ├─ Caster 连接 Runtime，注册 agents.execute rune
  ├─ 接收 RuneInput { config: AgentConfig, message: string }
  ├─ 根据 config 构建 pi-mono Agent:
  │   ├─ LLM: getModel(provider, model)
  │   ├─ Tools: bash (spawnHook 白名单) + read (allowed_paths)
  │   └─ SystemPrompt + constraints
  ├─ agent.prompt(message) → agent loop
  └─ 返回执行结果
```

## AgentConfig 格式

```typescript
interface AgentConfig {
  name: string;
  description: string;
  llm: {
    provider: string;
    model: string;
    max_tokens: number;
  };
  system_prompt: string;
  tools: {
    bash?: { allowed_binaries: string[] };
    read?: { allowed_paths: string[] };
    write?: { allowed_paths: string[] };
    edit?: { allowed_paths: string[] };
    grep?: {};
    find?: {};
    ls?: {};
  };
  constraints: {
    max_turns: number;
    timeout_secs: number;
  };
}
```

## 实施顺序

| Step | 内容 | 验证 |
|------|------|------|
| 1 | 项目骨架 + pi-mono 集成 | Agent 能跑通一轮 LLM + bash tool |
| 2 | Rune Caster 注册 | Caster 连上 Runtime，agents.execute rune 在线 |
| 3 | 端到端 | 通过 Runtime API 触发 agents.execute，完整跑通 |

## 安全

- 工具白名单：bash tool 通过 spawnHook 校验 allowed_binaries
- 文件约束：read/write/edit tool 通过 allowed_paths 限制
- 进程隔离：Caster 是独立 Node.js 进程

## 不做

- CLI（独立项目 my-agentos）
- 自定义工具 / 扩展工具
- 多 provider 验证（只验证 Anthropic）
- 容器化隔离 / MCP / Pipeline
