# Agent Caster v0.0.1 — Scope

在 microVM 沙箱中安全执行 Agent 任务，注册为 Rune Caster。

## 目标

`agents.execute` rune 端到端：接收 AgentConfig + message → 在 msb_krun microVM 中执行命令 → 返回结果。

## 架构

```
Rune Runtime (gRPC)
       │
Agent Caster ──→ SandboxBuilder ──→ msb_krun VM
       │                                  │
       │         virtio-console           │
       └──── AgentRelay ←───────→ Guest Agent (PID 1)
                                          │
                                    exec / fs ops
```

### Crate 职责

| Crate | 职责 |
|-------|------|
| `caster` | CLI 入口，Rune 注册，任务调度 |
| `sandbox` | Host 端 VM 管理 (SandboxBuilder → SandboxHandle) |
| `protocol` | Host↔Guest 线协议 (CBOR over virtio-console) |
| `guest-agent` | VM 内 PID 1，执行命令和文件操作 |
| `runner` | AgentConfig 解析 + ToolPolicy 生成 |

## AgentConfig

```yaml
name: feishu-assistant
description: "飞书助手"
llm:
  provider: anthropic
  model: claude-haiku-4-5-20251001
  max_tokens: 4096
system_prompt: |
  你是飞书助手。使用 feishu CLI 完成任务。
tools:
  bash:
    allowed_binaries: [feishu]
  read:
    allowed_paths: ["~/Documents/"]
constraints:
  max_turns: 10
  timeout_secs: 120
```

### Config → Sandbox 映射

```
tools.bash.allowed_binaries  →  Guest Agent exec 白名单
tools.read.allowed_paths     →  Volume mount (read-only)
tools.write.allowed_paths    →  Volume mount (read-write)
constraints.timeout_secs     →  VM idle timeout + exec timeout
```

## 实施顺序

| Step | 内容 | 验证 |
|------|------|------|
| 1 | protocol crate — 消息定义 + CBOR 编解码 | `cargo test -p agent-protocol` 序列化/反序列化通过 |
| 2 | guest-agent — PID 1 init + exec handler | 交叉编译成功，手动在 VM 内启动可执行命令 |
| 3 | sandbox crate — msb_krun VmBuilder + AgentRelay | `SandboxBuilder::new().create()` 能启动 VM 并收到 Ready |
| 4 | runner crate — AgentConfig 解析 + ToolPolicy | `cargo test -p agent-runner` YAML 解析通过 |
| 5 | caster — 串联 sandbox + runner | 本地 `handle_execute()` 跑通一次命令 |
| 6 | Rune 集成 | Caster 连上 Runtime，agents.execute rune 在线 |
| 7 | 端到端 | 通过 Runtime API 触发任务，sandbox 内执行，返回结果 |

## 安全模型

```
┌─────────────────────────────────────────┐
│  Hardware Virtualization (KVM / HVF)    │  ← 隔离边界
│  ┌─────────────────────────────────┐    │
│  │  Guest Linux Kernel             │    │
│  │  ┌───────────────────────────┐  │    │
│  │  │  Guest Agent (PID 1)      │  │    │
│  │  │  - exec 白名单校验        │  │    │
│  │  │  - 文件路径限制 (Volume)  │  │    │
│  │  └───────────────────────────┘  │    │
│  └─────────────────────────────────┘    │
└─────────────────────────────────────────┘
```

- 硬件级隔离：每个 Agent 任务在独立 microVM 中运行
- 二进制白名单：Guest Agent 在 exec 前检查 allowed_binaries
- 文件路径约束：只有 allowed_paths 作为 Volume 挂载进 VM
- 网络控制：TSI (Transparent Socket Impersonation) 或 airgapped
- 超时销毁：超时后 VM 直接 kill

## 不做

- LLM 集成 / Agent loop（v0.0.2，先跑通沙箱执行链路）
- 自定义 OCI 镜像构建
- Snapshot / Clone
- 多租户
- Web UI
