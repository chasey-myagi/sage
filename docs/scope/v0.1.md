# Agent Caster v0.1 — MVP Scope

第一个能跑的 AI Agent。端到端：接收任务 → 完整 Agent Loop 驱动 LLM → 7 个工具在沙箱中执行 → 流式返回结果。

## 目标

一句话验证：**Agent Caster = Pi 的大脑（全量移植） + 沙箱的身体**。

用户运行 `--local-test`，给一段自然语言任务，Agent 在 microVM 中自主推理执行，支持多轮 tool call、steering/followUp 注入、并行工具执行，流式输出过程，返回结果。

### 验收标准

```bash
# 基本多步任务
cargo run -p agent-caster -- --local-test \
    --config configs/coding-assistant.yaml \
    --message "在 /workspace 下创建一个 Rust hello world 项目并运行它"

# 验证 edit 工具
cargo run -p agent-caster -- --local-test \
    --config configs/coding-assistant.yaml \
    --message "在 /workspace 创建一个 Python 脚本打印 hello，然后用 edit 把 hello 改成 world，再运行验证"

# 验证 grep/find 工具
cargo run -p agent-caster -- --local-test \
    --config configs/coding-assistant.yaml \
    --message "在 /workspace 创建三个 .py 文件，然后用 find 找出所有 .py 文件，再用 grep 搜索其中包含 print 的文件"
```

预期行为：
1. Agent 调用 LLM（国产模型，如 Qwen）生成计划
2. 调用 bash/read/write/edit/grep/find/ls 工具在沙箱内执行
3. 支持并行工具调用（LLM 一次返回多个 tool_call 时）
4. 流式输出 agent_start → turn_start → message → tool_execution → turn_end → agent_end
5. steering/followUp 队列可通过 API 注入消息

## LLM Provider 策略

### 核心：OpenAI-Compatible Provider

国产大模型普遍兼容 OpenAI Chat Completions API。实现一个通用的 OpenAI-compatible provider，通过 `Model` 配置切换不同厂商：

| 厂商 | API 兼容 | baseUrl | 环境变量 |
|------|---------|---------|---------|
| Qwen (通义千问) | OpenAI Completions | `https://dashscope.aliyuncs.com/compatible-mode/v1` | `DASHSCOPE_API_KEY` |
| Doubao (豆包) | OpenAI Completions | `https://ark.cn-beijing.volces.com/api/v3` | `ARK_API_KEY` |
| Kimi (Moonshot) | OpenAI Completions | `https://api.moonshot.cn/v1` | `MOONSHOT_API_KEY` |
| MiniMax | OpenAI Completions | `https://api.minimax.chat/v1` | `MINIMAX_API_KEY` |
| z.ai (智谱) | OpenAI Completions | `https://open.bigmodel.cn/api/paas/v4` | `ZHIPU_API_KEY` |
| DeepSeek | OpenAI Completions | `https://api.deepseek.com` | `DEEPSEEK_API_KEY` |

**edit 工具 fuzzy match 细节（从 pi-mono edit-diff.ts 移植）：**

```rust
fn fuzzy_find_text(content: &str, old_text: &str) -> Option<FuzzyMatch> {
    // 1. 精确匹配
    if let Some(idx) = content.find(old_text) {
        return Some(FuzzyMatch::Exact(idx));
    }
    // 2. LF 归一化后匹配
    let norm_content = content.replace("\r\n", "\n");
    let norm_old = old_text.replace("\r\n", "\n");
    if let Some(idx) = norm_content.find(&norm_old) {
        return Some(FuzzyMatch::Normalized(idx));
    }
    // 3. Fuzzy 归一化：trailing whitespace + smart quotes + dashes + NFKC
    let fuzzy_content = normalize_for_fuzzy(&norm_content);
    let fuzzy_old = normalize_for_fuzzy(&norm_old);
    if let Some(idx) = fuzzy_content.find(&fuzzy_old) {
        return Some(FuzzyMatch::Fuzzy(idx));
    }
    None
}
```

**完整循环伪代码（Rust）：**

```rust
async fn run_loop(
    context: &mut AgentContext,
    new_messages: &mut Vec<AgentMessage>,
    config: &AgentLoopConfig,
    signal: CancellationToken,
    emit: &dyn AgentEventSink,
) -> Result<()> {
    let mut first_turn = true;
    let mut pending: Vec<AgentMessage> = config.get_steering_messages().await;

    loop {  // OUTER: follow-up
        let mut has_more_tool_calls = true;

        while has_more_tool_calls || !pending.is_empty() {  // INNER: steering + tools
            if !first_turn {
                emit(AgentEvent::TurnStart).await;
            }
            first_turn = false;

            // 1. 注入 pending messages（steering / follow-up）
            for msg in pending.drain(..) {
                emit(AgentEvent::MessageStart { message: &msg }).await;
                emit(AgentEvent::MessageEnd { message: &msg }).await;
                context.messages.push(msg.clone());
                new_messages.push(msg);
            }

            // 2. LLM 流式调用
            let assistant_msg = stream_assistant_response(
                context, config, &signal, emit
            ).await?;
            new_messages.push(AgentMessage::Assistant(assistant_msg.clone()));

            if matches!(assistant_msg.stop_reason, StopReason::Error | StopReason::Aborted) {
                emit(AgentEvent::TurnEnd { message: &assistant_msg, tool_results: &[] }).await;
                emit(AgentEvent::AgentEnd { messages: new_messages }).await;
                return Ok(());
            }

            // 3. 提取 tool calls
            let tool_calls = assistant_msg.tool_calls();
            has_more_tool_calls = !tool_calls.is_empty();

            // 4. 执行工具（并行或顺序）
            let tool_results = if has_more_tool_calls {
                let results = execute_tool_calls(
                    context, &assistant_msg, config, &signal, emit
                ).await?;
                for r in &results {
                    context.messages.push(AgentMessage::ToolResult(r.clone()));
                    new_messages.push(AgentMessage::ToolResult(r.clone()));
                }
                results
            } else {
                vec![]
            };

            emit(AgentEvent::TurnEnd { message: &assistant_msg, tool_results: &tool_results }).await;

            // 5. 轮询 steering 队列
            pending = config.get_steering_messages().await;
        }

        // 6. 检查 follow-up 队列
        let follow_ups = config.get_follow_up_messages().await;
        if !follow_ups.is_empty() {
            pending = follow_ups;
            continue;
        }

        break;
    }

    emit(AgentEvent::AgentEnd { messages: new_messages }).await;
    Ok(())
}

/// 工具执行三阶段（并行模式）
async fn execute_tool_calls_parallel(
    context: &AgentContext,
    assistant_msg: &AssistantMessage,
    tool_calls: &[ToolCall],
    config: &AgentLoopConfig,
    signal: &CancellationToken,
    emit: &dyn AgentEventSink,
) -> Result<Vec<ToolResultMessage>> {
    // Phase 1: prepare sequentially（emit start、validate args、beforeToolCall hook）
    let mut prepared = Vec::new();
    for tc in tool_calls {
        emit(AgentEvent::ToolExecutionStart { tool_call_id: &tc.id, tool_name: &tc.name, args: &tc.arguments }).await;
        match prepare_tool_call(context, assistant_msg, tc, config, signal).await {
            PrepareResult::Ready(p) => prepared.push(p),
            PrepareResult::Blocked(reason) => {
                // emit error result immediately
                emit_tool_error(tc, &reason, emit).await;
            }
        }
    }

    // Phase 2: execute concurrently
    let futures: Vec<_> = prepared.iter().map(|p| execute_prepared(p, signal, emit)).collect();
    let executed: Vec<_> = futures::future::join_all(futures).await;

    // Phase 3: finalize in assistant source order
    let mut results = Vec::new();
    for (prep, exec) in prepared.iter().zip(executed) {
        let result = finalize_tool_call(context, assistant_msg, prep, exec, config, signal, emit).await?;
        results.push(result);
    }

    Ok(results)
}
```

## 风险

| 风险 | 影响 | 缓解 |
|------|------|------|
| 国产 LLM tool_call 格式不一致 | 某些模型 tool call 返回格式有差异 | 先验证 Qwen/DeepSeek，兼容性适配层处理差异 |
| 国产 LLM thinking 格式碎片化 | reasoning_content 字段名/位置不统一 | ProviderCompat 抽象 + 按厂商适配 |
| edit fuzzy match 边界情况 | Unicode 规范化/BOM/CRLF 组合爆炸 | 从 pi-mono 移植已验证的逻辑，写充分的单元测试 |
| rg/fd 二进制在 VM 中不可用 | grep/find 工具无法执行 | rootfs 中预置 busybox grep/find 作为 fallback，或在 rootfs 构建时包含 rg/fd 静态二进制 |
| msb_krun macOS entitlements | 本地开发需要签名 | v0.0.1 已验证流程 |
| Agent Loop 死循环 | LLM 无限调工具 | max_turns 硬限制 + 单次 exec timeout + abort() |
| 大输出截断不当 | LLM 上下文爆炸 | 工具输出 2000 行 / 50KB 硬截断 |
